# JMM

* Java Memory Model Cookbook - JSR 133. Она не описывает текущую модель памяти, которая гарантируется текущими компиляторами.

* Visibility
* Reordering (компиляторные перестановки, процессорные перестановки)
* Happens Before (отношения < на операциях означающая, что если X < Y, то все инструкц до X видны везде после Y)

Happens Before гарантирует то, что мы после Y видим (visibility) всё, что было сделано до X

* JCStress для тестирования многопоточных программ

* Multiple Atomicity (все процессоры либо не видят ничего либо видят все) -- такого в Java нет (такой гарантии!)
* Нельзя мыслить терминами локальных кэшей потока и глобальной памяти
